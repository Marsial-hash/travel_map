# swifthire.top 参考分析

> 分析日期：2026-05-10
> 来源：[swifthire.top](https://swifthire.top/)
> 网站性质：AI 面试助手（通用技术岗），Vue.js + Element Plus 构建

## 整体定位

面向技术岗求职者的 AI 面试辅助平台，核心卖点是**AI 驱动**：AI 预测面试题、AI 模拟面试官。

## 功能模块

| 模块 | 说明 | 对我们的参考价值 |
|------|------|----------------|
| **简历押题** | 上传简历 → AI 从题库匹配可能的面试题 | ★★★★★ 与我们的"简历面"思路一致，差异化在于我们用财务题库 |
| **AI 模拟面试** | AI 面试官实时对话、动态追问、评分报告 | ★★★★☆ 方向好但技术门槛高，MVP 阶段可以先做"脚本式" |
| **八股检索** | 按公司/年份/难度/轮次搜题 | ★★★★☆ 面经库的搜索功能可参考 |
| **面经来源** | 2500+ 篇真实面经 | ★★★★★ 与我们的面经库完全一致 |

## 数据展示参考

首页数据概览设计：
- 8000+ 真实面经题目
- 2500+ 篇面经来源
- 2048 维向量检索
- 3 大面试环节

## 导航结构

```
首页 | 简历预测 | 八股检索 | 更多 → 我的预测/我的检索/订单核销/数据看板
```

## 可借鉴的点

1. **简历→出题流程**：上传简历 → 提取关键词 → 匹配题库 → 展示预测题，逻辑清晰
2. **模拟面试的对话式 UI**：一问一答的聊天界面，有面试官角色和候选人角色区分
3. **分类检索**：按公司、轮次、难度、年份多维度筛选
4. **数据卡片**：首页用数字大标题展示平台数据，增强信任感

## 差异化方向（我们怎么做更好）

- 专注**财务/审计**垂直领域，不做大而全
- 面经来源可以做成结构化归档，便于复习
- 我们的优势是用户可以自己导入面经（Word/PDF/图片），形成私有面经库

## 2026-05 v0.3 架构调整：参考 swifthire 后

参考 swifthire 的"简历预测→出题→练习"串联流程后，将原本独立的**简历制作**和**模拟面试**两个模块**合并为"简历押题"**：

```
上传简历 → 自动解析 → 展示简历摘要 → AI 预测面试题
    ↓
点击题目 → 进入对话练习（面试官 ↔ 候选人）
    ↓
修改简历 → 一键重新生成题目
```

这一设计参考了 swifthire 的**简历→出题→练习**串联逻辑，但在财务垂直领域做了适配：分类改为行为面/简历面/专业面，题目内容聚焦财务审计场景。
